# gsm_project_cluster (Resource)

Registers a game server cluster to a project.

## Prerequisites

The game server cluster must already exist (created via `gsm_game_server_cluster` resource).

## Dependency Chain

The cluster registration follows this dependency order:

1. Create `gsm_game_server_cluster` (standalone cluster)
2. Create `gsm_project_cluster` (register to project) - **this resource**
3. Create `gsm_environment_cluster` (register to environment)

## Labels and Annotations

You can define labels and annotations for the cluster registration at the project level. How these relate to environment-level cluster registrations is handled by the backend API.

## Deletion Considerations

If you need to remove a cluster from the project:

- Ensure the cluster is not registered to any environments in this project first
- The backend API will reject the deletion if there are dependent environment cluster registrations

## Example Usage

```terraform
resource "gsm_project_cluster" "example" {
  project_id   = gsm_project.example.project_id
  cluster_name = gsm_game_server_cluster.example.cluster_name

  labels = {
    environment = "production"
    team        = "platform"
  }

  annotations = {
    description = "Main production cluster"
  }
}

# Import example:
# terraform import gsm_project_cluster.example gsm_project_cluster/<project_id>/<cluster_name>
```
