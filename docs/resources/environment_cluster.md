# gsm_environment_cluster (Resource)

Registers a game server cluster to an environment.

## Prerequisites

Before creating an `gsm_environment_cluster` resource, ensure:

1. The game server cluster exists (created via `gsm_game_server_cluster`)
2. The cluster is registered to the project (created via `gsm_project_cluster`)
3. The environment belongs to the same project

## Registration Behavior

Registering a cluster to an environment **replaces** any existing registration with the same cluster name.

- Each registration is independent - you specify labels and annotations for this environment
- If you register the same cluster again, it completely overwrites the previous registration
- How the backend API uses project cluster settings is handled server-side, not by Terraform

## Example Usage

```terraform
resource "gsm_environment_cluster" "example" {
  environment_id = gsm_environment.example.environment_id
  cluster_name   = gsm_game_server_cluster.example.cluster_name

  labels = {
    region = "us-west-2"
    tier   = "production"
  }

  annotations = {
    description = "Production environment cluster"
  }
}

# Import example:
# terraform import gsm_environment_cluster.example gsm_environment_cluster/<environment_id>/<cluster_name>
```
